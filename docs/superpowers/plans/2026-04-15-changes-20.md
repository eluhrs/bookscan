# CHANGES-20 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Consolidate the edit page and workflow Review step onto a single shared `BookCard` component driven by an `editable` prop, with a redesigned field layout (typography hierarchy per FEAT-02), inline-editable Year + Publisher, hide-when-empty additional fields, and a + button on the Review filmstrip.

**Architecture:** One new presentational component `frontend/src/components/BookCard.tsx` owns the filmstrip + field block + condition/review toggle rows + description block, plus an additional-fields section gated on `editable`. It encapsulates its own `DraftFields` + `InlineField` (moved over from `BookEditCard.tsx`). The edit page (`DashboardPage` edit branch) renders `<BookCard editable />` inside its existing `position:fixed` viewport shell; `ReviewStep` renders `<BookCard editable={false} />` inside `WorkflowWrapper`. Both existing consumers (`BookEditCard.tsx` + the field block in `ReviewStep.tsx`) are retired once the new component is proven. No backend changes are required — `BookUpdate` already accepts `year` and `publisher`.

**Tech Stack:** React 18 + TypeScript strict, Vite, `@zxing/browser` (unchanged), FastAPI + SQLAlchemy 2.0 async (untouched), Vitest + jsdom. Theme tokens in `frontend/src/styles/theme.ts`. Lucide icons.
