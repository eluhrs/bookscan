**Goal:** Fix horizontal scroll on camera screens, unify border contrast, make the dashboard responsive on mobile, and remove the Condition column from desktop.

**Tech Stack:** React, TypeScript, inline styles, Lucide icons, CSS media queries via injected `<style>` tag.

### Task 1: FIX-12 — Prevent horizontal scroll on camera workflow screens

**Files:**
- Modify: `frontend/src/components/workflow/WorkflowWrapper.tsx:70-86` (outer container)

The outer `position: fixed` container already has `overflow: hidden`, but the camera `<video>` elements inside PhotographStep and LookupStep use `width: 100%` which can exceed bounds. The fix is to add `overflowX: 'hidden'` to the outer container (belt-and-suspenders) and ensure `maxWidth: '100vw'` on the container.

- [ ] **Step 1: Add maxWidth constraint to WorkflowWrapper outer container**

In `frontend/src/components/workflow/WorkflowWrapper.tsx`, add `maxWidth: '100vw'` to the outer fixed container style (around line 78):

```tsx
style={{
  position: 'fixed',
  top: 0,
  left: 0,
  right: 0,
  maxWidth: '100vw',
  height: vpHeight,
  transform: `translateY(${vpOffset}px)`,
  background: theme.colors.surface,
  color: theme.colors.text,
  display: 'flex',
  flexDirection: 'column',
  overflow: 'hidden',
  fontFamily: theme.font.sans,
}}
```

- [ ] **Step 2: Verify Review step and keyboard step are unaffected**

Review step has no `<video>` — it renders a filmstrip and form fields. Keyboard step renders an `<input>` with `maxWidth: 320`. Neither should be affected by the `maxWidth: '100vw'` constraint. No changes needed for those paths.

### Task 3: FEAT-01 — Mobile-responsive dashboard table

**Files:**
- Modify: `frontend/src/components/BookTable.tsx` (add responsive columns, icon buttons, row click handler)

The approach: inject a `<style>` tag with CSS media queries to hide columns on small screens. Each `<td>` and `<th>` gets a className for its column. On mobile (`max-width: 767px`), only status icons, title, and action icons are visible. Action buttons become Lucide `Pencil` and `Trash2` icons. Row click on mobile navigates to edit.

- [ ] **Step 1: Add Pencil and Trash2 imports**

In `frontend/src/components/BookTable.tsx`, update the lucide import:

```tsx
import { FileWarning, Camera as CameraIcon, Pencil, Trash2 } from 'lucide-react'
```

- [ ] **Step 2: Inject responsive CSS via `<style>` tag**

Add a `<style>` block at the top of the returned JSX (inside the outer `<div>`), before the `<table>`:

```tsx
<style>{`
  @media (max-width: 767px) {
    .bt-col-author,
    .bt-col-publisher,
    .bt-col-year { display: none !important; }
    .bt-col-actions-text { display: none !important; }
    .bt-col-actions-icon { display: table-cell !important; }
  }
  @media (min-width: 768px) {
    .bt-col-actions-text { display: table-cell !important; }
    .bt-col-actions-icon { display: none !important; }
  }
`}</style>
```

- [ ] **Step 3: Add classNames to existing columns**

Add `className` to each `<th>` and `<td>` that needs hiding:
- Author th/td: `className="bt-col-author"`
- Publisher th/td: `className="bt-col-publisher"`
- Year th/td: `className="bt-col-year"`

For the `colHeader` function, add an optional `className` parameter:

```tsx
function colHeader(label: string, key: SortKey, className?: string) {
  // ... existing code ...
  return (
    <th
      className={className}
      onClick={() => handleSort(key)}
      // ... rest unchanged
    >
```

Then update calls:
```tsx
{colHeader('Title', 'title')}
{colHeader('Author', 'author', 'bt-col-author')}
{colHeader('Publisher', 'publisher', 'bt-col-publisher')}
{colHeader('Year', 'year', 'bt-col-year')}
```

Add `className="bt-col-author"` / `bt-col-publisher` / `bt-col-year` to the corresponding `<td>` elements in the row map.

- [ ] **Step 4: Replace text action buttons with two columns — text actions (desktop) and icon actions (mobile)**

Replace the single Actions `<th>/<td>` with two:

Desktop actions th/td (`className="bt-col-actions-text"`): keep existing List/Edit/Delete text buttons.

Mobile actions th/td (`className="bt-col-actions-icon"`): two icon buttons:

```tsx
<th className="bt-col-actions-icon" style={{ display: 'none', padding: '0.6rem 0.5rem', width: 72 }} />
```

```tsx
<td className="bt-col-actions-icon" style={{ display: 'none', padding: '0.6rem 0.5rem', whiteSpace: 'nowrap' }}>
  <button
    onClick={(e) => { e.stopPropagation(); onEdit(book) }}
    style={{
      marginRight: '0.5rem',
      padding: '0.35rem',
      border: `1px solid ${theme.colors.border}`,
      borderRadius: theme.radius.sm,
      background: theme.colors.bg,
      cursor: 'pointer',
      display: 'inline-flex',
      alignItems: 'center',
      justifyContent: 'center',
    }}
    aria-label="Edit book"
  >
    <Pencil size={16} color={theme.colors.muted} />
  </button>
  <button
    onClick={(e) => { e.stopPropagation(); onDelete(book.id) }}
    style={{
      padding: '0.35rem',
      border: `1px solid ${theme.colors.danger}`,
      borderRadius: theme.radius.sm,
      background: theme.colors.bg,
      cursor: 'pointer',
      display: 'inline-flex',
      alignItems: 'center',
      justifyContent: 'center',
    }}
    aria-label="Delete book"
  >
    <Trash2 size={16} color={theme.colors.danger} />
  </button>
</td>
```

- [ ] **Step 5: Add row click handler for mobile**

Add `onClick` to each `<tr>` in the tbody:

- [ ] **Step 6: Update colSpan on empty state row**

The "No books yet" empty row uses `colSpan={7}`. After adding the mobile actions column, there are now 7 visible columns at most (status + title + author + publisher + year + desktop-actions + mobile-actions), but since one actions column is always hidden, the effective max is 6. Use `colSpan={99}` to safely span all columns regardless of visibility:

```tsx
<td colSpan={99} style={{ ... }}>
```
